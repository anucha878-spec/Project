พื้นฐานร่วม

Flow เดียวกันทั้ง 4 batch — เข้าใจอันนี้ก่อน ที่เหลือดูแค่ “จุดต่าง”

ดึงกลุ่มเป้าหมาย→ คัด Do Not Contact→ map policy_no → cardNo→ ตรวจ LINE/App via cisappapi→ dedup กันส่งซ้ำ→ สร้าง CSV (UTF-8)→ ESB → SMS Gateway→ log (S/F) + refer_no→ email เมื่อรอบล้มเหลว→ Manual Fix

ข้อยุติ Error/Recovery ร่วม (inherited จาก CSMS-001)

  • E13 / ผลผิดปกติรายรายการ → skip + บันทึก F ไม่ retry ไม่หยุดรอบ
  • API ล่มทั้งระบบ → retry ≤3 ครั้ง แล้ว หยุดรอบ + email
  • รายการ F → daily ไม่ retry อัตโนมัติ ซ่อมผ่าน Manual Fix เท่านั้น
  • Manual Fix = idempotent หยิบเฉพาะ sms_sent_date IS NULL

Endpoint / โครงสร้างกลางที่ใช้ร่วม

  • ตรวจสถานะ: cisappapi @ 11.100.8.44
  • ส่ง: Web Service กลาง ESB → SMS Gateway (คืน refer_no)
  • DNC: ili_policy_remark · Config: cf_catalog
  • ไฟล์: MKT_CSMS_[Category]_[YYMMDDhhmmss].csv (ปี พ.ศ. 2 หลัก)

💡 QA tip: เขียน test suite “Shared Behavior” ชุดเดียว (error/log/Manual-Fix/idempotency) แล้ว reuse ข้าม batch — ลด TC ซ้ำ

CSMS-001   branch 002-...line-sms

Welcome New Customer (LINE) Daily 10:00

◆ Business

  • ต้อนรับลูกค้าใหม่ที่กรมธรรม์ Inforce เมื่อวาน (T-1) ปี 2026+
  • ยังไม่ลงทะเบียน LINE Ocean Connect / Ocean Club
  • เป้าหมาย: ยืนยันความคุ้มครอง + ชวนลง LINE

▼ Input

  • กรมธรรม์ ORD / IND / PA · issue_date = T-1 · Inforce · มี mobile_no
  • Source: ili_application_osp_ord/ind/pa, ili_policy_master, ili_customer_data_analytics
  • DNC: ORD remark_code='1' · IND/PA ='4' (map O/I,G/P)

⚙ Config

  • SMS_CATEGORY code 113 = MKTWelcomeNewCust
  • ลิงก์: cf_catalog type=LINE_LINK, value1=CSMS_SNC_NewCust, active='Y'
  • template: sms_message_schedule → fallback SMS_CATEGORY

▶ Output

  • CSV MKT_CSMS_MKTWelcomeNewCust_...csv
  • ตาราง WELCOME_NEW_CUST_LINE (ใหม่) + CSMS_LOG
  • รายงานประจำวัน (US3) · email เมื่อล้มเหลว
Dedup: รายลูกค้า (fname,lname,mobile) → MIN(issue_date) ส่งเมื่อ: ไม่พบ LINE+Club | ไม่มี Club+block LINE ไม่ส่ง: พบ LINE(ไม่บล็อก) หรือมี Club
CSMS-002   branch 003-...app-sms

Welcome New Customer รอบ 2 — Ocean Club App Daily 10:00 · ~1,000/วัน

◆ Business

  • Follow-up ต่อจากรอบ 1 เมื่อครบ 20 วัน (ปรับได้)
  • ลูกค้ายังไม่ลง LINE + App → ชวนโหลด Ocean Club App + ของรางวัล

▼ Input

  • อ่านจากประวัติรอบ 1: WELCOME_NEW_CUST_LINE.sms_sent_date + N วัน
  • กรมธรรม์ ORD/IND/GOV/PA Inforce (ORD/PA='1'; IND/GOV='1','2','3')
  • เบอร์ = ปัจจุบันจากกรมธรรม์ (mobile1→mobile2) ไม่ใช่เบอร์รอบแรก

⚙ Config

  • N วัน: cf_catalog type=DATE_COUNT (CSMS_SNC_NewCustApp, default 20)
  • รางวัล REWARDS (var2) · ลิงก์ LINE_LINK (var3)
  • template code 111

▶ Output

  • CSV MKT_CSMS_MKTWelcomeNewCustApp_...csv
  • ตาราง WELCOME_NEW_CUST_APP (ใหม่) — มี FK อ้างกลับรอบ 1
  • รายงานกลาง CSMS (ไม่แยกใหม่)
Dedup: รายลูกค้า 1 ข้อความ/รอบ ⚠ GOV รวมสโคป (ต่างจาก 001) ⚠ FR-020.2 ปน Manual-Fix + reporting
CSMS-003   branch 004-gp7-sms-batch

Grace Period 7 วันก่อนหมดผ่อนผัน Daily 10:00 · LINE-only

◆ Business

  • เตือนกรมธรรม์ Inforce เลย due_date แต่ยังใน grace period
  • เหลือ 7 วันปฏิทิน ก่อน grace_end_date · เฉพาะยังไม่ลง LINE
  • เป้าหมาย: กระตุ้นชำระเบี้ยก่อนขาดความคุ้มครอง

▼ Input

  • ORD/IND/PA Inforce · grace_end_date − current_date = 7 (ไม่รวมวันปัจจุบัน)
  • Source: ili_policy_master + ili_notification_letter
  • ตรวจ LINE เท่านั้น (ไม่ตรวจ App)

⚙ Config

  • จำนวนวัน: cf_catalog type=REMINDER_DAYS (CSMS_GP_7Days, default 7)
  • template Grace Period · vars: var1=policy_no, var2=due_date, var3=grace_end_date

▶ Output

  • CSV MKT_CSMS_MKTGracePeriod7Days_...csv
  • ตาราง SMS_GRACE_PERIOD_LOG (ใหม่) · ฟิลด์ sms_sent_status
🔑 Dedup รายกรมธรรม์: key = policy_no + due_date → 2 กรมธรรม์ = 2 ข้อความ ส่ง: E02 | คัดออก: E00+isBlockLine=false NFR: ≤5 นาที / 10,000 รายการ
CSMS-004   branch 005-batch-birthday-sms

Customer Birthday Daily 09:00 (ต่างจากทุก batch)

◆ Business

  • อวยพรวันเกิด (วัน/เดือนตรงวันประมวลผล) + ของขวัญ + ชวนลง LINE
  • เฉพาะลูกค้า ช่องทางตัวแทน ที่ยังไม่ใช้ทั้ง LINE + App

▼ Input

  • ORD/IND/GOV/PA Inforce · birthdate วัน/เดือน = วันนี้ (ไม่เทียบปี)
  • เบอร์ coalesce(mobile1,mobile2) · Source: ili_policy_master, ili_agent_master
  • Agent-only: ORD title_desc≠'ชำระเอง' · อื่น position_code≠'99' · policy_org ∈ 2070001–2079999 / 5070001–5079999

⚙ Config

  • SMS_CATEGORY code 112 = MKTBirthday
  • ของขวัญ REWARDS (CSMS_SBD_Birthday) · ลิงก์ LINE_LINK (CSMS_SBD_Birthday)
  • template code 112 · vars: var1=fname, var2=ของขวัญ, var3=ลิงก์

▶ Output

  • CSV MKT_CSMS_MKTBirthday_...csv — คอลัมน์เฉพาะ: mobile,var1,var2,var3,seq_no,datetime
  • ตาราง CUSTOMER_BIRTHDAY (ใหม่) + CSMS_LOG
🔑 Dedup รายลูกค้า (cardNo) 1 ข้อความ/วันเกิด/ปี ตรวจ 2 ช่องทาง: LINE=E02 และ App=E02 → ส่ง 29 ก.พ. ปีไม่อธิกสุรทิน → ส่ง 28 ก.พ.
