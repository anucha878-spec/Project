V2 Advantage · ของที่ได้เพิ่ม

🟣 จุดที่ V2 เหนือกว่า

🧬 Atom-inventory (00b)

  • ย่อย FR → testable atom เป๊ะ (error text/boundary/enum)
  • audit atom→TC ไม่ใช่แค่ FR→TC
  • FR ครบ 100% ≠ atom ครบ

❓ RQ ยิงกลับ SA

  • field ไม่ระบุ = RQ + severity (ไม่เดา)
  • RQ-002 ปี CSV พ.ศ. vs ค.ศ.
  • RQ dedup ต้อง key 'S' เท่านั้น (CRITICAL)

📊 ปลายสายครบ

  • RTM reconcile (0 orphan/dangling)
  • 08 report + GO/NO-GO verdict
  • NFR-routing (PII/DNC track)

🤖 Automation-ready

  • Playwright *.spec.js ฝัง TC-ID
  • blocked-stage stub บอกวิธีปลดล็อก
  • redmine = dry-run (ไม่สร้าง ticket)
Global Advantage · ของที่ได้เพิ่ม

🟢 จุดที่ Global เหนือกว่า

⚡ กระชับ / อ่านไว

  • 7 ไฟล์/แคมเปญ · flat
  • ไม่มี stub เปล่า
  • เหมาะรีวิวเร็ว

📋 CSV tracker

  • test-cases_*.csv 12 คอลัมน์
  • 16-col checklist ตรง format บริษัท
  • หยิบเข้า SIT ได้ทันที

👀 เข้าใจง่าย

  • ไม่ต้องรู้ spec-kit numbering
  • โครงสร้างตรงไปตรงมา
  • สำหรับคนไม่คุ้นระบบ

Global เก็บ TC เป็น CSV/checklist เป็นหลัก จึง import เข้าเครื่องมือ SIT/tracker ได้เร็วกว่า ขณะที่ V2 เก็บ TC เป็น .md เป็นหลัก

Bottom line · สรุป

V2 ครบและลึกกว่า · Global เบาและเร็วกว่า

ทั้งคู่หยุดที่กำแพงเดียวกัน (ไม่มีแอปให้ execute) และให้ verdict บนฐานเดียวกัน — ต่างกันตรงที่ V2 เขียน verdict นั้นออกมาเป็นเอกสารส่งต่อได้ ขณะที่ Global ไปไม่ถึง stage report

🔓 ปลดล็อกเพิ่มได้เมื่อ

  • ส่ง path เอกสารต้นทาง BRD/FRD/UR → เปิด crosscheck
  • มี SIT env + batch harness → เปิด execute → report → regression
