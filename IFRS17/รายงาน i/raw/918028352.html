    วัตถุประสงค์

    (Objective)

    1. เพื่อ Export รายงาน I01-I05 เพื่อขอเลข ICG/RCG Tagging จาก IFRS 17

    2. เพื่อ Update ICG/RCG Tagging ที่ฐานข้อมูล Monthly Policy Status Snapshot

    3. เพื่อ Export รายงาน I01-I05 ข้อมูลกรมธรรม์พร้อม Tagging เพื่อส่งข้อมูลให้ IFRS 17

    4. เพื่อแสดงรายละเอียดสถานะการประมวลผล

    5. เพื่อยกเลิกรายการนำเข้าข้อมูล

    ผู้ใช้งาน
    (Target Users)

    ฝ่ายคณิตศาสตร์

    3

    ข้อมูลที่ป้อนสู่ระบบ

    (Input)

    1. กระบวนการนำส่งไฟล์เพื่อติด Tagging: ข้อมูลกรมธรรม์เคสใหม่หรือกรมธรรม์ที่ยังไม่มี ICG/RCG Tagging

    2. กระบวนการนำเข้าไฟล์ Tagging: ข้อมูล ICG/RCG Tagging

    3. กระบวนการนำส่งข้อมูล: ข้อมูลกรมธรรม์ และข้อมูล ICG/RCG Tagging

    4

    ข้อมูลที่ได้จากระบบ

    (Output)

    1. ข้อมูล Monthly Policy Status Snapshot 

      1. ข้อมูลกรมธรรม์หลัก ประกอบด้วยข้อมูลกรมธรรม์, ข้อมูลลูกค้า

      2. ข้อมูล Rider ประกอบด้วยข้อมูลกรมธรรม์ Rider และข้อมูลลูกค้า

      3. ข้อมูล Tagging

    2. รายงาน I01-I05 โดยเป็นข้อมูลกรมธรรม์ และ ICG/RCG Tagging

    3. รายละเอียดการประมวลผลข้อมูล

    การกระทำกับหน้าจอ

    4. ยกเลิกรายการนำเข้าข้อมูล

    5. ดูรายละเอียดการประมวลผล

    อธิบายรายละเอียด

    (Description)

    Screen MPS002-FD001-SD001 - หน้าจอ ประมวลผล MPS รายละเอียด ดังนี้

    1. หน้าจอแสดงรายการข้อมูล ณ รอบการประมวลผลของรอบเดือนปัจจุบัน พร้อมสถานะรายการล่าสุด ประกอบด้วยข้อมูล ดังนี้

      1. Dropdown “ประจำเดือน” แสดงเดือน ปีของรอบการประมวลผลปัจจุบัน โดยแสดงผลแบบ “ปี/เดือน” (YYYY/MM) โดยมีรูปแบบดังนี้

        1. เดือน ในรูปแบบ MM ตัวอย่างเช่น 02

        2. ปี เป็นปี พ.ศ. ในรูปแบบ YYYY ตัวอย่างเช่น 2565

        3. ระบบสามารถนำเข้าข้อมูลได้เฉพาะรอบการประมวลผลปัจจุบัน
        4. สามารถแสดงประวัติการประมวลผลย้อนหลังได้สูงสุด 5 เดือน

        5. กรณีพบประวัติการประมวลผลย้อนหลัง ระบบแสดงดังนี้

          • ระบบแสดงข้อมูลของรอบการประมวลผลที่เลือกจาก dropdown “ประจำเดือน”

          • Disable ปุ่ม “ยกเลิกรายการ”, “นำส่งไฟล์เพื่อติด Tagging”, “นำเข้าไฟล์ Tagging”, “ส่งข้อมูลเข้าระบบ EDW”

          • Enable ปุ่ม “รายละเอียด” 

        6. กรณีไม่พบประวัติการประมวลผลย้อนหลัง ระบบแสดงดังนี้

          • ระบบแสดง “ไม่พบข้อมูล”

          • ไม่แสดงข้อมูลของ “สถานะรายการ”, “Shared Drive Folder”

          • Disable ปุ่ม “รายละเอียด”, “ยกเลิกรายการ”, “นำส่งไฟล์เพื่อติด Tagging”, “นำเข้าไฟล์ Tagging”, “ส่งข้อมูลเข้าระบบ EDW”

             ตัวอย่าการแสดงหน้าจอกรณีไม่พบประวัติการประมวลผลย้อนหลัง

        7. เพื่อรองรับการทดสอบ SIT, UAT สำหรับการกำหนดรอบประมวลผลย้อนหลัง หรือล่วงหน้า จึงมีการปรับให้กำหนดค่า รอบประมวลผล จาก system configure จากเดิม default ที่เดือนปัจจุบัน - 1 โดยมีเงื่อนไขตาม A17. การกำหนดรอบประมวลผล

      2. “สถานะรายการ” แสดงสถานะการประมวลผล (อ้างอิง สถานะ จาก Appendix A10.3 สถานะรายการหน้าจอ ประมวลผล MPS)

        1. ดึงค่าสถานะจาก tx_mps_process_header.mps_process_status_id ด้วยเงื่อนไข period = รอบประมวลผลเดือนปัจจุบัน และหากพบมากกว่า 1 รายการ ให้ระบบแสดงรายการล่าสุด

        2. กรณีไม่พบ Transaction ใน tx_mps_process_header ของรอบประมวลผลปัจจุบัน หรือพบแต่ Transaction ล่าสุดมีสถานะเป็น "ยกเลิกรายการ" ให้ระบบแสดงสถานะการประมวลผลเป็น "รอนำเข้าข้อมูล"

      3. "Shared Drive Folder " แสดงชื่อของ Shared Drive Folder ของรอบการประมวลผลปัจจุบัน โดยแสดงผลแบบ “เดือน ปี (Path)” ดึงค่าจาก tx_mps_process_header.shared_drive_folder

        1. กำหนดให้ Path ขึ้นต้นด้วย ServerIP

        2. ตัวอย่างเช่น 202212 (ServerIP/IFRS17/Report I)

      4. ปุ่ม “รายละเอียด” สำหรับดูรายละเอียดการประมวลผล  หลังกดปุ่ม ระบบดำเนินการเปิดหน้าจอ “Popup รายละเอียด” (อ้างอิง Screen SD002 หน้าจอ Popup รายละเอียด)

      5. ปุ่ม “ยกเลิกรายการ” สำหรับการยกเลิกนำเข้าข้อมูลในรอบประมวลผลปัจจุบัน โดยระบบดำเนินการ ดังนี้

        1. เปิดหน้าจอ Popup ยืนยันการยกเลิกนำเข้าข้อมูล พร้อมให้ระบุหมายเหตุ

        2. หลังกดปุ่ม “ยืนยัน” ระบบดำเนินการยกเลิกข้อมูลนำเข้าก่อนหน้า

          1.  บันทึกค่าสถานะใน tx_mps_process_header เป็น "ยกเลิกรายการ"

          2. หน้าจอแสดงสถานะการประมวลผลเป็น รอนำเข้าข้อมูล

          3. insert Log process ที่ตาราง tx_mps_process_history เป็น "ยกเลิกรายการ"

          4. ตรวจสอบข้อมูล master ฝั่ง core ในรอบประมวลผลที่ทำรายการ หากพบข้อมูลให้ลบข้อมูลด้วยเงื่อนไขรอบประมวลผลนั้น ๆ: (tx_mps_master_i01(CoreEDW), tx_mps_master_i02 (CoreEDW), tx_mps_master_i03 (CoreEDW), tx_mps_master_i04 (CoreEDW), tx_mps_master_i05 (CoreEDW)) เพื่อรองรับกรณีที่ต้องการส่งข้อมูลเข้า edw อีกครั้งในรอบประมวลผล
