กำหนดให้ $รอบประมวลผลปัจจุบัน มีค่าเท่ากับ ปีเดือนปัจจุบัน - 1 เดือน โดยให้ดึงค่าจาก [A17. การกำหนดรอบประมวลผล](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=971014593)

แต่สำหรับการทดสอบ SIT, UAT สามารถปรับให้รอบประมวลผล ย้อนหลัง หรือล่วงหน้าได้มากกว่า 1 เดือน

>
**1. Update ข้อมูลกรมธรรม์เก่า ที่ตาราง tx_mps_master_i04**

>
**1.1 บันทึกข้อมูลกรมธรรม์**

1. **ดึงข้อมูลกรมธรรม์จากตาราง Landing **[tx_mps_landing_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_landing_i04)** จากกระบวนการสิ้นเดือน** ด้วยเงื่อนไข Period = $รอบประมวลผลปัจจุบัน เดือนปัจจุบัน
2. กรณีมีการขอ RCG Tagging เพิ่มเติม ดึงข้อมูลส่วน RCG ตาราง [tx_mps_landing_rcg](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_landing_rcg) ด้วยเงื่อนไข Period = $รอบประมวลผลปัจจุบัน เดือนปัจจุบัน และ policy_number = [tx_mps_landing_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_landing_i04).policy_number
3. **นำข้อมูลจากข้อ 1, 2 มา Update ข้อมูลจากตาราง Landing ที่ตาราง Master **[tx_mps_master_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_master_i04)** **เงื่อนไขการ Update ข้อมูลอ้างอิงจาก Column "Landing" จาก [EDW-MPS-PS004-02 เงื่อนไขการบันทึกข้อมูลกรมธรรม์ ยูนิต ลิงค์](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=930840830)

>
**1.2 ลบข้อมูลส่วน RI สำหรับบางกรมธรรม์ที่มีการปรับปรุงข้อมูลจากกระบวนการ Actual Account**

>
**1.2.1 หารายการที่เข้าเงื่อนไข**

1. ดึงข้อมูลกรมธรรม์ที่มีข้อมูล RI ในรอบเดือนปัจจุบันจาก [tx_ri_support_bk_dt](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_dt) [tx_ri_support_bk_oic](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic) ด้วยเงื่อนไขดังนี้ period = $รอบประมวลผลปัจจุบัน
2. policy_number = เลขที่กรมธรรม์
3. report_type มีค่าเป็น 'New', 'Renew', 'New(Fac)', 'Renew(Fac)' โดย convert ให้เป็น UPPERCASE ทั้งหมดทั้ง Key และ Value
4. code_plan = แบบประกัน
5. movement มีค่า 'R' = 'D'
6. ri_std_hd_id = [tx_ri_std_hd](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_std_hd_id และต้องเป็น header ที่มีสถานะนำเข้าสำเร็จ ([tx_ri_std_hd](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_process_hd_id = [tx_ri_process_header](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_process_hd_id และ [tx_ri_process_header](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_process_status_id = 3) tx_ri_std_hd_oic.ri_std_hd_id และต้องเป็น header ที่มีสถานะนำเข้าสำเร็จ (tx_ri_std_hd_oic.ri_process_hd_id = tx_ri_process_header_oic.ri_process_hd_id และ tx_ri_process_header_oic.ri_process_status_id = 3)
7. tx_ri_spbk_oic_hd [tx_ri_support_bk_oic_hd](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic_hd).coi_flag = True (ปรับเนื่องจาก New Closing OIC)

**1.3 Update ส่วนข้อมูล Reinsurance เพิ่มเติม**

>
**1.3.1 ดึงข้อมูลจาก support booking**

1. ดึงข้อมูลกรมธรรม์ที่มีข้อมูล RI ในรอบเดือนปัจจุบันจาก [tx_ri_support_bk_dt](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_dt) [tx_ri_support_bk_oic](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic) ด้วยเงื่อนไขดังนี้ period = $รอบประมวลผลปัจจุบัน
2. policy_number = [tx_mps_master_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_master_i04).policy_number
3. report_type มีค่าเป็น 'New', 'Renew', 'New(Fac)', 'Renew(Fac)' โดย convert ให้เป็น UPPERCASE ทั้งหมดทั้ง Key และ Value
4. code_plan = [tx_mps_master_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_master_i04).plan_code
5. movement ไม่ใช่ค่า 'R' กับ 'D' เพื่อกรองข้อมูล RI ที่ถูกลบตอนกระบวนการ RI actual (อ้างอิงกระบวนการ RI actual: [MPS_userProcessFlow-RI Process.drawio.png](http://wiki.thaisamut.co.th/download/attachments/946143409/MPS_userProcessFlow-RI%20Process.drawio.png?version=1&modificationDate=1654074155085&api=v2))
6. ri_std_hd_id = [tx_ri_std_hd](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_std_hd_id และต้องเป็น header ที่มีสถานะนำเข้าสำเร็จ ([tx_ri_std_hd](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_process_hd_id = [tx_ri_process_header](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_process_hd_id และ [tx_ri_process_header](http://wiki.thaisamut.co.th/display/RDSADW/Process+Generate+I04+Untag).ri_process_status_id = 3) tx_ri_std_hd_oic.ri_std_hd_id และต้องเป็น header ที่มีสถานะนำเข้าสำเร็จ (tx_ri_std_hd_oic.ri_process_hd_id = tx_ri_process_header_oic.ri_process_hd_id และ tx_ri_process_header_oic.ri_process_status_id = 3)
7. tx_ri_spbk_oic_hd [tx_ri_support_bk_oic_hd](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic_hd).coi_flag = True (ปรับเนื่องจาก New Closing OIC)
ตรวจสอบค่าเบี้ยของ RI (ri_premium) ว่า เป็น 0 หรือไม่
1. กรณีค่าเบี้ยของ RI **เป็น** 0 ถือว่า ไม่เข้าเงื่อนไขการส่งประกันภัยต่อ ดำเนินการดังนี้ไม่เข้าเงื่อนไขในข้อ **1.3.2 หากพบรายการเข้าเงื่อนไขตามด้านบน ให้ Update ข้อมูล**** **และ** (PH4.1) Update ข้อมูล RI NAR**
2. เข้าเงื่อนไขในข้อ **1.4 Clear RCG Tagging กรณีไม่พบข้อมูล Reinsurance **และดำเนินการตามข้อถัดไป
3. กรณีผลรวม **ไม่เป็น** 0 ถือว่า เข้าเงื่อนไขการส่งประกันภัยต่อ ดำเนินการตามข้อถัดไป
นำเลขกรมธรรม์ และ Treaty_code ไปตรวจสอบหาเงื่อนไขการ update ข้อมูลค้นหาข้อมูลใน [tx_mps_master_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_master_i04) ด้วยเงื่อนไข เลขกรมธรรม์ และหาเงื่อนไขการ update ข้อมูลดังนี้
>
**เงื่อนไขที่ 1: Update Treaty Code เดิม หากเป็นข้อมูล Reinsurer ที่เคยมีข้อมูลใน Master อยู่แล้ว**

หาก [tx_ri_support_bk_dt](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_dt) [tx_ri_support_bk_oic](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic).treaty_code จากข้อ 2. ตรงกับใน treaty_code_1 หรือ treaty_code_2 หรือ treaty_code_3 ให้ **Update ข้อมูลตาม Treaty Code นั้นๆ**

>
**เงื่อนไขที่ 2: Update Treaty Code ที่เป็นค่าว่าง หากเป็นข้อมูล Reinsurer ที่ยังไม่เคยบันทึกข้อมูลใน Master**

หาก [tx_ri_support_bk_dt](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_dt) [tx_ri_support_bk_oic](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic).treaty_code จากข้อ 2. ไม่ตรงกับค่าใน treaty_code_1 หรือ treaty_code_2 หรือ treaty_code_3 ให้ **Update ข้อมูลใน ****treaty_code_1 หรือ treaty_code_2 หรือ treaty_code_3 ที่เป็นค่าว่างตามลำดับ**

ตัวอย่าง หาก [tx_ri_support_bk_dt](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_dt) [tx_ri_support_bk_oic](http://wiki.thaisamut.co.th/display/RDSADW/tx_ri_support_bk_oic).treaty_code ไม่ตรงกับ treaty_code_1 แต่ treaty_code_2 เป็นค่าว่าง ให้ update ค่าใน treaty_code_2
