# A17. การกำหนดรอบประมวลผล

- url: http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=971014593
- http status: 200
- source: Playwright MCP (in-body link, depth 1)

---
เพื่อรองรับการทดสอบ SIT, UAT สำหรับการกำหนดรอบประมวลผลย้อนหลัง หรือล่วงหน้า จึงมีการปรับให้กำหนดค่า รอบประมวลผล จาก system configure จากเดิม default ที่เดือนปัจจุบัน - 1 เดือน

>
**กระบวนการคำนวณ Period สำหรับประมวลผล**

**การกำหนดรอบประมวลผล คำนวณจาก **

1. **$รอบประมวลผลเริ่มต้น** : กำหนดเดือนที่ทำการดึงข้อมูลเพื่อตั้งฐาน MPS
2. **$ประมวลผลเกินงวดปัจจุบัน** : กำหนดว่าจะสามารถทำการประมวลผล MPS เกินกว่ารอบประมวลผลเดือนปัจจุบัน (current date) - 1 เดือน

**การกำหนดรอบประมวลผล**สำหรับตั้งฐาน

1. กระบวนการตั้งฐาน จะใช้วันที่ **$รอบประมวลผลเริ่มต้น** เป็นตัวกำหนดว่าจะดึงข้อมูลเพื่อตั้งฐาน MPS
2. เมื่อกระบวนการตั้งฐาน เสร็จสิ้นจนถึงกระบวนการ **ยืนยันข้อมูลเข้าระบบ EDW **จะถือว่ากระบวนการตั้งฐานเสร็จเรียบร้อย
3. กรณียังตั้งฐานไม่เสร็จเรียบร้อย หน้าจอต่างๆ ที่แสดงข้อมูลรอบการประมวลผล จะแสดงเป็น **$รอบประมวลผลเริ่มต้น**

**การกำหนดรอบประมวลผลสำหรับ Ongoing**

1. รอบประมวลผลสำหรับ Ongoing จะคิดจาก **$รอบประมวลผลล่าสุด ที่เสร็จเรียบร้อย** โดยมีกรณีต่างๆ ดังนี้กรณีเป็นรอบประมวลผลต่อ Ongoing ที่ต่อจากรอบการ **$รอบประมวลผลเริ่มต้น** + **1 เดือน**
2. กรณีเป็นรอบประมวลผลรอบถัดไป จะใช้ **$รอบประมวลผลล่าสุด ที่เสร็จเรียบร้อย**
การกำหนด **$ประมวลผลเกินงวดปัจจุบัน**
1. กรณี **$รอบประมวลผลล่าสุด **ถึงเดือนปัจจุบัน และ** $ประมวลผลเกินงวดปัจจุบัน**** **= True ระบบจะ **อนุญาต **ให้กดปุ่ม Execute เพื่อดึงข้อมูลใน**หน้าจอ Batch Process (Manual) **สำหรับประมวลผลเกินรอบประมวลผลเดือนปัจจุบัน (current date) - 1 เดือน
2. กรณี **$รอบประมวลผลล่าสุด **ถึงเดือนปัจจุบัน และ** $ประมวลผลเกินงวดปัจจุบัน**** **= False ระบบจะ **ไม่อนุญาต **ให้กดปุ่ม Execute เพื่อดึงข้อมูลใน**หน้าจอ Batch Process (Manual) ในกรณีที่ประมวลผลเกินรอบประมวลผลเดือนปัจจุบัน (current date) - 1 เดือน******
3. กรณี **$ประมวลผลเกินงวดปัจจุบัน** **= ****False** ระบบจะนำ **$รอบประมวลผลล่าสุด** เทียบกับ วันที่ปัจจุบัน หากเปลี่ยนเดือนแล้วระบบถึงจะทำการเปลี่ยน period โดยนำ **$รอบประมวลผลล่าสุด** **+1 เดือน**
4. กรณี **$ประมวลผลเกินงวดปัจจุบัน** **= True** ระบบจะนำ **$รอบประมวลผลล่าสุด** **+1 เดือน** โดยไม่สน วันที่ปัจจุบัน
5. กรณี **หน้าจอ Batch Manual** ระบบไม่สนใจ **$ประมวลผลเกินงวดปัจจุบัน **ระบบจะนำ **$รอบประมวลผลล่าสุด** **+1 เดือน** โดยไม่สน วันที่ปัจจุบัน
