ปุ่ม **“นำเข้าไฟล์ Tagging”** สำหรับนำเข้าไฟล์ข้อมูล Tagging จาก Landing Path และ Update ICG/RCG Tagging รายละเอียด ดังนี้

1. เมื่อกดปุ่ม ระบบแสดงสถานะการประมวลผลเป็น “กำลังดำเนินการนำเข้าไฟล์ Tagging”
2. insert Log process ที่ตาราง [tx_mps_process_history](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_process_history)
3. ดำเนินการนำเข้าไฟล์ข้อมูล Tagging จาก Shared Path
4. บันทึกสถานะการนำเข้าไฟล์ Tagging แต่ละไฟล์เป็น "กำลังดำเนินการ" ที่ตาราง [tx_mps_import_icg](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_import_icg)
5. ตรวจสอบไฟล์นำเข้า รายละเอียดดังนี้
6. ตรวจสอบชื่อไฟล์ถูกต้องตาม Format และเป็นรูปแบบไฟล์ CSV และไม่ใช่ไฟล์เปล่า อ้างอิงรูปแบบชื่อไฟล์จาก [A3.1 การกำหนดรูปแบบไฟล์นำเข้า Tagging](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=917471603) กรณีไม่พบไฟล์นำเข้า หรือชื่อไฟล์ไม่ตรงตาม Format หรือเป็นไฟล์เปล่าที่ไม่มีข้อมูลในไฟล์ ให้ระบบแสดงสถานะการนำเข้าไฟล์ Tagging เป็น "นำเข้าไม่สำเร็จ" พร้อมแสดงหมายเหตุ “ไม่พบข้อมูลไฟล์ต้นทาง”
7. กรณีชื่อไฟล์ถูกต้องตาม Format ดำเนินการข้อถัดไป

ตรวจสอบรูปแบบไฟล์ เป็น .CSV

1. กรณีไม่ใช่ .CSV ระบบแสดงสถานะการนำเข้าไฟล์ Tagging เป็น "นำเข้าไม่สำเร็จ" พร้อมแสดงหมายเหตุ เป็น “Invalid File Format”
2. กรณีรูปแบบไฟล์ถูกต้อง ดำเนินการข้อถัดไป

ตรวจสอบ จำนวนคอลัมน์ ของไฟล์นำเข้าว่าถูกต้องหรือไม่

1. หากไม่ถูกต้อง ระบบปรับสถานะรายการเป็น "รูปแบบไฟล์ไม่ถูกต้อง" พร้อมแสดงหมายเหตุ "จำนวน Column ไม่ถูกต้องรบกวนตรวจสอบรูปแบบของไฟล์" โดยไม่แสดง Error File และจบกระบวนการทำงาน
2. หากถูกต้อง ให้ดำเนินการตรวจสอบในข้อถัดไป

ตรวจสอบ หัวคอลัมน์ ของไฟล์นำเข้าว่าถูกต้องหรือไม่

1. หากไม่ถูกต้อง ระบบปรับสถานะรายการเป็น "รูปแบบไฟล์ไม่ถูกต้อง" พร้อมแสดงหมายเหตุ "ชื่อ Column ไม่ถูกต้องรบกวนตรวจสอบรูปแบบของไฟล์" โดยไม่แสดง Error File และจบกระบวนการทำงาน
2. หากถูกต้อง ให้ดำเนินการตรวจสอบในข้อถัดไป
ตรวจสอบหากพบข้อมูลกรมธรรม์มีข้อมูลไม่ครบถ้วน (เนื่องจากระบบต้องรองรับกรณีที่นำเข้ามาเป็นไฟล์เปล่า มีข้อมูลเฉพาะ header)

1. กรณีไฟล์ ICG Tagging หรือ RCG Tagging หากพบข้อมูลในไฟล์มีข้อมูลไม่ครบถ้วน มีค่าใดเป็นค่าว่าง ระบบแสดงสถานะการนำเข้าไฟล์ Tagging เป็น "นำเข้าไม่สำเร็จ" พร้อมแสดงหมายเหตุเป็น “พบ Null Value” และ Generate Error File ในรูปแบบ Excel File เพื่อใช้ในการตรวจสอบข้อมูล **โดยแสดงเฉพาะ row no และข้อมูลของรายการที่ error** (อ้างอิง Error File จาก [A3.1 การกำหนดรูปแบบไฟล์นำเข้า Tagging](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=917471603))
2. กรณีไฟล์ ICG/RCGG Tagging หากพบข้อมูลในไฟล์มีข้อมูลไม่ครบถ้วน อ้างอิงการตรวจสอบ nullable จาก [A3.1.3 รูปแบบเอกสารไฟล์นำเข้า ICG/RCG Tagging](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=918028547) ระบบแสดงสถานะการนำเข้าไฟล์ Tagging เป็น "นำเข้าไม่สำเร็จ" พร้อมแสดงหมายเหตุเป็น “พบ Null Value” และ Generate Error File ในรูปแบบ Excel File เพื่อใช้ในการตรวจสอบข้อมูล **โดยแสดงเฉพาะ row no และข้อมูลของรายการที่ error** (อ้างอิง Error File จาก [A3.1 การกำหนดรูปแบบไฟล์นำเข้า Tagging](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=917471603))
3. กรณีข้อมูลครบถ้วน ไม่พบค่าว่าง ดำเนินการข้อถัดไป

ตรวจสอบจำนวนรายการของไฟล์นำเข้า Tagging ที่ Landing Path ไม่เท่ากับจำนวนรายงาน I01-I05

2. ไฟล์นำเข้า RCG Tagging: กรณี I01, I03, I04: นำเลขที่กรมธรรม์ "Policy Number" และ "Prophet Plan Code" เทียบกับค่าเลขที่กรมธรรม์ "policy_number" และ "prophet_plan_code" จากตาราง [tx_mps_untag_i01](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_untag_i01), [tx_mps_untag_i03](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_untag_i03),[tx_mps_untag_i04](http://wiki.thaisamut.co.th/display/RDSADW/tx_mps_untag_i04) เฉพาะรายการที่มีค่าในฟิลด์ treaty_code_1
