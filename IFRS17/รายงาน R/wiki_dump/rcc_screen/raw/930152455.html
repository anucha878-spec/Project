(Description)

EDW-RCC-SD002 หน้าจอแสดงรายละเอียด และยืนยันการออกรายงาน R

  1. รูปแบบการแสดงช่วงวันที่ประมวลผลข้อมูล เป็น Text แสดง ปี และเดือน ที่ได้รับจากหน้าจอ EDW-RCC-SD001 หน้าจอประมวลผลข้อมูล และค้นหาข้อมูล โดยมีรูปแบบการแสดงผลเป็น: YYYY/MM กำหนดให้ YYYY เป็นปี ค.ศ. พ.ศ. และ MM เป็นเดือน

  2. รูปแบบการแสดงช่วงวันที่ประมวลผลข้อมูล เป็น Text แสดง ปี และเดือน ที่ได้รับจากหน้าจอ EDW-RCC-SD001 หน้าจอประมวลผลข้อมูล และค้นหาข้อมูล โดยมีรูปแบบการแสดงผลเป็น: YYYY/MM กำหนดให้ YYYY เป็นปี ค.ศ. และ MM เป็นเดือน

  3. สถานะ ใช้ข้อมูลจาก EDW-RCC-SD001 หน้าจอประมวลผล และค้นหาข้อมูล ในส่วนของสถานะดำเนินการ มีดังนี้
    1. รอยืนยันข้อมูล หมายถึง เมื่อกดปุ่มประมวลผล และประมวลผลสำเร็จมีข้อมูลครบถ้วนไม่เกิดข้อผิดพลาด เพื่อให้ User สามารถ Download Excel File เพื่อตรวจเช็ค และยืนยันข้อมูลในลำดับถัดไป
    2. ประมวลผลไม่สำเร็จ หมายถึง เมื่อกดปุ่มประมวลผลแล้ว เกิดข้อผิดพลาดจากระบบ เช่น ระบบไม่สามารถเชื่อมต่อฐานข้อมูลได้
      หรือ Connection lost
    3. ยืนยันข้อมูลเรียบร้อย หมายถึง ผู้ใช้ได้ทำการยืนยันข้อมูลทั้งหมด เรียบร้อยแล้ว
    4. ยกเลิกการประมวลผล หมายถึง เมื่อมีสถานะรอยืนยันข้อมูล และผู้ใช้พบว่าข้อมูลไม่ถูกต้อง และกดยกเลิกการประมวลผล
  4. วันที่ เวลา ประมวลผล แสดงวันที่ ที่ผู้ใช้ทำการกดปุ่มประมวลผลที่ EDW-RCC-SD001 หน้าจอประมวลผล และค้นหาข้อมูล มีรูปแบบเป็น: DD/MM/YYYY HH:MM:SS โดยที่ YYYY เป็น ปี ค.ศ. พ.ศ. ตัวอย่าง 30/06/2564 13:00:00
  5. ข้อมูล "รวมจำนวนเงิน" หมายถึง จำนวนเงินรวมทั้งหมดของรายงาน R01-R20
  6. ข้อมูล Shared Drive Folder หมายถึง ให้แสดงชื่อ Folder ตามข้อมูลที่จัดเก็บ

  7. รายละเอียดของข้อมูล Data grid มีดังต่อไปนี้

    1. การยืนยันข้อมูล โดยเลือก Checkbox จะยืนยันข้อมูลได้ก็ต่อเมื่อระบบมีสถานะดำเนินการเป็น "ประมวลผลสำเร็จ" เท่านั้น
    2. Reconcile & adjustment ใช้สำหรับ Download file reconcile and adjustment posting โดยแยกตามประเภทกลุ่มข้อมูล จะสามารถ Download ได้ก็ต่อเมื่อระบบมีสถานะดำเนินการเป็น "ประมวลผลสำเร็จ" หรือ "ยืนยันข้อมูล" เท่านั้น (กรณีที่เป็นกลุ่มของรายงาน R13 ให้ดูตัวอย่างที่ รายงาน EDW-RCC-RP013 สำหรับออกรายงาน R13 - Ordinary Products [OP] Expense Allocation GMM)
    3. รายงาน R ใช้สำหรับ Download file รายงาน R โดยแยกตามประเภทกลุ่มข้อมูล จะสามารถ Download ได้ก็ต่อเมื่อระบบมีสถานะดำเนินการเป็น "ประมวลผลสำเร็จ" หรือ "ยืนยันข้อมูล" เท่านั้น
    4. ประเภทกลุ่มข้อมูล แสดงประเภทกลุ่มข้อมูล อ้างอิง A2. Group Type
    5. จำนวนเงิน แสดงจำนวนเงิน adjustment posting
    6. สถานะดำเนินการ มีดังนี้

      1. ประมวลผลสำเร็จ หมายถึง เมื่อกดปุ่มประมวลผลแล้ว ข้อมูลครบถ้วนไม่เกิดข้อผิดพลาด เพื่อให้ User สามารถ Download Excel File เพื่อตรวจเช็ค และยืนยันข้อมูลในลำดับถัดไป

      2. ประมวลผลไม่สำเร็จ หมายถึง เมื่อกดปุ่มประมวลผลแล้ว เกิดข้อผิดพลาดจากระบบ เช่น ระบบไม่สามารถเชื่อมต่อฐานข้อมูลได้

    7. วันที่ยืนยัน/ยกเลิกข้อมูล แสดงวันที่ ที่ผู้ใช้ทำการยืนยันข้อมูล มีรูปแบบเป็น: DD/MM/YYYY กำหนดให้ DD เป็นวันที่, YYYY เป็นปี ค.ศ. พ.ศ. และ MM เป็นเดือน ตัวอย่าง 30/06/2564
    8. ผู้ยืนยัน/ยกเลิกข้อมูล แสดงชื่อผู้ดำเนินการยืนยันข้อมูล โดยแยกตามแต่ละประเภทกลุ่มข้อมูล
  8. ปุ่ม "ยืนยันข้อมูล" เมื่อผู้ใช้ทำการประมวลผลข้อมูลสำเร็จ และเลือก Check box ใน Grid view จะสามารถกดปุ่มยืนยันข้อมูลได้ โดยเมื่อกดปุ่มระบบจะแสดง Popup โดยมีรายละเอียดดังนี้
    1. ส่วนของ Dropdownlist จะแสดงข้อความให้เลือกเพื่อ ยืนยันข้อมูล หรือ ยกเลิกการประมวลผล
    2. ผู้ใช้สามารถระบุหมายเหตุได้
    3. กรณีที่เลือกยืนยันข้อมูลและกดบันทึกระบบจะ Update สถานะรายการที่ผู้ใช้เลือก และจัดเก็บหมายเหตุ(กรณีที่ระบุ)
      1. ระบบทำการส่ง Email แจ้งการยืนยันข้อมูล (ตัวอย่าง Email EDW-RCC-AM001 แจ้งผลดำเนินการ ออกรายงาน R ตามช่วงการประมวลผล)
    4. กรณีที่ยืนยันข้อมูลครบทุกรายการ ให้ระบบทำการ Rename folder ที่อยู่ใน Share drive โดยเพิ่มข้อความ Approved (ตัวอย่างเช่น Approved_202203_to_202207_20220721144259) และให้ทำการ Update ชื่อ Folder ในตารางที่จัดเก็บข้อมูล
    5. กรณีที่เลือกยกเลิกการประมวลผล ระบบจะ Require ให้ผู้ใช้ระบุหมายเหตุ ก่อนการกดบันทึก
      1. กรณีที่กดบันทึก และระบุผลการตรวจสอบข้อมูลเป็น ยกเลิกการประมวลผล ระบบจะปรับสถานะทุกตัวที่มีสถานะเป็น รอยืนยันข้อมูลให้เป็นยกเลิกการประมวลผลทั้งหมด แต่หมายเหตุจะขึ้นอยู่กับรายการที่ผู้ใช้เลือก Checkbox เท่านั้น
      2. ระบบทำการส่ง Email แจ้งการยกเลิกการประมวลผล (ตัวอย่าง Email EDW-RCC-AM001 แจ้งผลดำเนินการ ออกรายงาน R ตามช่วงการประมวลผล)
  9. ปุ่ม "Back" ใช้สำหรับย้อนกลับไปยังหน้าจอ EDW-RCC-AM001 แจ้งผลดำเนินการ ออกรายงาน R ตามช่วงการประมวลผล และคงเงื่อนไขการค้นหาเดิมไว้ แต่สถานะและข้อมูลใน Grid view จะต้องเปลี่ยนไปตามข้อมูลที่ถูก Update

  10. ปุ่ม "สร้างไฟล์ SAP" ใช้สำหรับ Generate Text File SUN จากข้อมูล Adjustment Posting ของ R-Report (R01-R20) โดยมีเงื่อนไขดังนี้
    1. ปุ่ม "สร้างไฟล์ SAP"
      1. Enable (สามารถกดปุ่มได้) ก็ต่อเมื่อทุกรายการมีสถานะ "ยืนยันข้อมูล" และยังไม่เคยกดสร้างไฟล์ SUN โดยจะแสดงเป็นปุ่มสีฟ้า
      2. Disable (ไม่สามารถกดปุ่มได้) ก็ต่อเมื่อมีรายการใดรายการหนึ่ง ไม่เป็นสถานะ "ยืนยันข้อมูล" หรือมีการกดสร้างไฟล์ SUN ไปแล้ว โดยจะแสดงเป็นปุ่มสีเทา
    2. กรณีที่กดปุ่ม ระบบจะแสดง Popup ยืนยันการ Generate Text File SUN

      1. กรณีกดปุ่ม "ตกลง" ระบบจะทำการเรียก Process Generate Text File SUN อ้างอิงรายละเอียด Process การสร้างข้อมูล File SUN จากข้อมูล Adjustment Posting ของ R-Report (R01-R20)
      2. กรณีกดปุ่ม "ยกเลิก" ระบบทำการปิด pop-up
  11. ปุ่ม "กลับรายการ SAP" ใช้สำหรับ การสร้างข้อมูล กลับรายการ Generate Text File SAP โดยมีเงื่อนไขดังนี้
    1. ปุ่ม กลับรายการ SAP
      1. Enable (สามารถกดปุ่มได้) ก็ต่อเมื่อเคยมีการกดสร้างไฟล์ SUN แล้ว และยังไม่เคยมีการกดกลับรายการ SUN โดยจะแสดงเป็นปุ่มสีฟ้า  
      2. Disable (ไม่สามารถกดปุ่มได้) ก็ต่อเมื่อยังไม่เคยมีการกดสร้างไฟล์ SUN แล้ว หรือเคยมีการกดกลับรายการ SUN แล้ว โดยจะแสดงเป็นปุ่มสีเทา
    2. กรณีที่กดปุ่ม ระบบจะแสดง Popup ยืนยันการกลับรายการ SUN
      1. กรณีกดปุ่ม "ตกลง" ระบบจะทำการเรียก Process กลับรายการ SUN อ้างอิงรายละเอียด Process การสร้างข้อมูล File SUN จากข้อมูล Adjustment Posting ของ R-Report (R01-R20)
      2. กรณีกดปุ่ม "ยกเลิก" ระบบทำการปิด pop-up
  12. Download file ทั้งหมด เมื่อกดปุ่มให้ระบบดำเนินการ Download file รายงาน RCC01-RCC20 รวม One page summary และ ZIP File โดยมีเงื่อนไขดังนี้ --UR 20250074: EDW PH5.1 เพิ่ม function download file ทั้งหมด (07.05.2025)
    1. Enable ปุ่มกด ก็ต่อเมื่อพบรายการ File RCC ให้สามารถ Download ได้ 1 รายการเป็นต้นไป
      1. File ที่สามารถ Download ได้จะต้องไม่มีสถานะเป็นประมวลผลไม่สำเร็จ (เงื่อนไขตามเดิม เหมือนการ Download แยกทีละ File)
      2. แสดง Popup เพื่อให้ผู้ใช้ยืนยันการ Download file ทั้งหมด
      3. กรณีตอบยืนยันให้ระบบทำการ Download file ทั้งหมด เป็นรูปแบบ ZIP โดยมี Format: R01_R20ACC_YYYYMM_to_YYYYMM.zip
      4. กรณียกเลิก ให้ระบบทำการปิด Popup
    2. Disable (ไม่สามารถกดปุ่มได้) ก็ต่อเมื่อพบรายการ File RCC ไม่สามารถ Download ได้ทุกรายการ โดยจะแสดงเป็นปุ่มสีเทา
