⭐ Test Focus

9จุดต้องทดสอบหนัก (Critical / Boundary)

  • SLA Timer (TS-17): Hot นับจาก 'เวลานัด' · Warm นับจาก 'เวลา assign' ในช่วง 9–18 น. → ทดสอบขอบ 07:00 (เริ่ม 9:00), 18:00 (พอดี), 19:00 (เริ่ม 9:00 วันถัดไป), ครบ 24 ชม.พอดี
  • Dedup (TS-12/09): 1 ช่องทาง vs 2 ช่องทาง · แบบประกันเหมือน/ต่าง · ยึด CIS เป็นหลัก · Lead ซ้ำ → LC คนเดิม
  • PDPA/Compliance (TS-04/09): Consent=No และ Do-not-contact ต้องถูก exclude ทั้งตอนสร้าง Segment และตอนส่งจริง
  • Maker/Checker (TS-03/08): แยกบทบาท — Maker อนุมัติงานตัวเองไม่ได้ · Reject ต้องมีเหตุผล
  • File Upload (TS-02): boundary 5.00MB ยอมรับ / 5.01MB ปฏิเสธ · column ผิด · ไฟล์ว่าง · ไทย UTF-8
  • Counter (TS-06/10): Sent=สำเร็จเท่านั้น · Unique Click ไม่นับซ้ำ · Open แยกจาก Click
  • Import LC (TS-11): boundary ≤10 ชื่อ/จังหวัด และ ≤10 จังหวัด/ครั้ง (10 ยอมรับ / 11 ปฏิเสธ)

11GrowthAi เปลี่ยนวิธีทดสอบ Campaign

GrowthAi คือเครื่องมือ marketing-automation จริง — Campaign ไม่ใช่ "สร้าง+เลือกช่องทาง" แต่เป็น Visual Flow Builder

[Segment / Contact] → [Decision] → [Condition ตาม tag] → [Action] yes/no branch ├─ Send email (drag-drop builder) ├─ Send text message ├─ Mobile app push ├─ Send webhook └─ per-step delay / schedule
ต้องทดสอบ: แต่ละ node แยก + ทั้ง flow แบบ E2E + combinatorial path (yes/no × branch × action) + per-step delay
Tracking ตายตัวต่อ campaign: Total User · Total Send · Total Credit · Total Fail · Total Click · Unique Click (+ drill-down รายคน)

❓ Clarify First

13ต้องถามให้ชัดก่อนเขียน Expected

  • SLA 'ครบ 24 ชม.พอดี' นับ inclusive/exclusive? และนับข้ามวันหยุด (เสาร์-อาทิตย์) หรือนับ 24 ชม.ต่อเนื่อง? (TC-17-03/07/13)
  • GrowthAi เป็น COTS/SaaS หรือ custom build? — กระทบระดับการทดสอบ (config vs functional) + effort
  • Send webhook ยิงไประบบไหน (MCMP internal / LC Connect / UW?) — ยังไม่อยู่ใน BRD
  • Decision vs Condition node ต่างกันเชิง business อย่างไร · LINE Ocean Connect ส่งผ่าน action ใด (หายจาก GrowthAi)
  • Total Credit / Total Fail สูตรตรงกับ Infobip billing หรือไม่ (reconciliation)
  • การแก้ Campaign หลังอนุมัติ — ล็อก หรือ ต้องขออนุมัติใหม่? (TC-08-05)
  • Raw Data Lead Source แสดงหลายช่องทางด้วย comma หรือไม่ (Issue #105 Recheck) · frequency sync RFM (daily vs monthly)
ข้อที่ค้างคำตอบ → ทำ TC เป็น 'Blocked/Draft' ไว้ก่อน อย่าเดา Expected — ถาม BA/SA (piraya.ph) ให้จบก่อนรัน

17Test Execution Workflow & Defect Flow

เตรียม ─► Import TC ─► รัน P1 ก่อน ─► บันทึกผล ─► เจอ Defect ─► แจ้ง Dev ─► Retest ─► ปิด (Test Data) (Jira/Xray) (ตาม chain) (Pass/Fail) (ระบุ TC+step) (severity) (verify) (closed)
Entry Criteria (เริ่มเทสต์ได้เมื่อ):
build พร้อม · test data เตรียมครบ · integration/stub เชื่อมได้ · Open Issue High ได้คำตอบ
Exit Criteria (จบรอบได้เมื่อ):
P1 ทั้งหมดรัน 100% + ผ่าน · ไม่มี Critical/Blocker ค้าง · defect P2 มีแผนปิด · trace RTM ครบ
Defect report ต้องอ้าง TC ID + step + expected/actual + severity + test data ที่ใช้ เพื่อให้ dev reproduce ได้ทันที
