- จากนั้นนำ ms_product.ms_product_id เทียบกับ cf_reserve_rate_hd.ms_product_id ตรวจสอบที่ cf_reserve_rate_hd.aprv_status ของทุกรายการ ต้องเท่ากับ "APRV" เท่านั้น เฉพาะกรมธรรม์ที่เป็น policy_type = ORD และ IND กรณีที่มีรายการใดรายการหนึ่ง ไม่เท่ากับ "APRV" ให้หยุดการประมวลผลของ Treaty นั้น จากนั้นบันทึกสถานะ tx_ri_est_hd.status = ERROR และ บันทึก tx_ri_est_hd.remark = "มีรายการ Reserve Rate ที่ยังไม่ได้อนุมัติ กรุณาตรวจสอบข้อมูล" กรณีที่ทุกรายการเท่ากับ "APRV" ให้ดำเนินการต่อ

1. **[เก่า]**ตรวจสอบรายการชุดข้อมูลที่จะต้องนำไปประมวลผล (Current_Policy_New,Current_Policy_Renew,Current_Policy_Renew_RI)ตรวจสอบ Effective Date From - To ว่า Period อยู่ในช่วงการประมวลผลหรือไม่กรณีอยู่ในช่วงการประมวลผล (Treaty ยังไม่หมดอายุ)Current_Policy_New กรมธรรม์ใหม่ตรงตามรอบCurrent_Policy_Renew กรมธรรม์เก่าที่เคยส่งแล้วส่งต่อ ยังไม่หมดอายุCurrent_Policy_Renew_RI กรมธรรม์ใหม่ที่ไม่เคยส่งมาก่อน เพราะฉะนั้นจะมีได้ทั้ง New และ Renewกรณีไม่อยู่ในช่วงการประมวลผล (Treaty หมดอายุแล้ว)กรณี Config ให้ส่งประกันต่อที่ Renew Alter ClaimCurrent_Policy_Renew กรมธรรม์เก่าที่เคยส่งแล้วส่งต่อ ยังไม่หมดอายุCurrent_Policy_Renew_RI กรมธรรม์ใหม่ที่ไม่เคยส่งมาก่อน เพราะฉะนั้นจะมีได้ทั้ง New และ Renew กรณี Config ไม่ต้องส่งประกันต่อใด ๆ ให้ข้ามการประมวลผล Treaty นั้นไป มีผลเป็น 0 กรมธรรม์ทุก ๆ Sheet
2. **[ใหม่]**ตรวจสอบกรมธรรม์ต้องไม่ถูก Mark ไว้ว่าไม่ต้องส่งประกันต่อ
3. **[เก่า]**ตรวจสอบวันที่เริ่มต้นสัญญาของกรมธรรม์ ต้องอยู่ในช่วง Effective Date - From To ของข้อมูลหลัก Treatyถ้าวันที่เริ่มต้นสัญญาของกรมธรรม์ ไม่ตรงกับช่วงที่ Effective Date - From To ของข้อมูลหลัก Treaty ไม่ต้องนำไปประมวลผลต่อถ้าวันที่เริ่มต้นสัญญาของกรมธรรม์ ตรงกับช่วงที่ Effective Date - From To ของข้อมูลหลัก Treaty ให้นำไปตรวจสอบข้อต่อไป
4. **[ใหม่]**ตรวจสอบวันที่เริ่มต้นสัญญา Rider Code ทุกตัวแยกกัน (เนื่องจากวันที่เริ่มต้นสัญญา Rider อาจะมีการเปลี่ยนแปลง ดังนั้นจะแทนค่าด้วย #วันที่เริ่มต้นสัญญาเดิม และ #วันที่เริ่มต้นสัญญาใหม่)เปรียบเทียบข้อมูลวันที่เริ่มต้นสัญญา Rider Code ระหว่าง #วันที่เริ่มต้นสัญญาเดิม และ #วันที่เริ่มต้นสัญญาใหม่ถ้า #วันที่เริ่มต้นสัญญาเดิม มีค่า แต่ #วันที่เริ่มต้นสัญญาใหม่ ไม่มีค่า ให้ใช้ #วันที่เริ่มต้นสัญญาเดิม ไปตรวจสอบต่อถ้า #วันที่เริ่มต้นสัญญาเดิม มีค่า และ #วันที่เริ่มต้นสัญญาใหม่ มีค่าวันที่เริ่มต้นสัญญาเดิม เท่ากับ #วันที่เริ่มต้นสัญญาใหม่ ให้ใช้ #วันที่เริ่มต้นสัญญาเดิม ไปตรวจสอบต่อวันที่เริ่มต้นสัญญาเดิม น้อยกว่า #วันที่เริ่มต้นสัญญาใหม่ ให้ใช้ #วันที่เริ่มต้นสัญญาใหม่ ไปตรวจสอบต่อ
5. **[ใหม่]**ตรวจสอบการกำหนด Treaty ที่ต้องตรวจสอบรายการเคลมกรณี Treaty ถูกกำหนดให้ต้องตรวจสอบกรณีกำหนดเป็นแบบที่ 1 ตรวจสอบจาก Rider Code ที่จะนำไปประมวลผลส่ง New Renew ต่อ กับ Percent Claimถ้า Percent Claim ของ Rider นั้นครบ 100% แล้ว ไม่ต้องนำไปประมวลผลต่อถ้า Percent Claim ของ Rider นั้น ยังไม่ครบ 100% ให้นำไปตรวจสอบข้อต่อไปกรณีกำหนดเป็นแบบที่ 2 ตรวจสอบกลุ่มความคุ้มครองโรคร้ายแรงถ้า Percent Claim ของกลุ่มโรคร้ายแรงกลุ่มใดครบ 100% แล้ว ไม่ต้องนำไปประมวลผลต่อถ้า Percent Claim ของกลุ่มโรคร้ายแรงกลุ่มใดยังไม่ครบ 100% ให้นำไปตรวจสอบข้อต่อไปกรณี Treaty ไม่ถูกกำหนดให้ตรวจสอบ ให้ข้ามไปตรวจสอบข้อต่อไป
6. **[ใหม่]**ตรวจสอบการประมวลผลในรูปแบบ Package โดยให้ตรวจสอบในกลุ่ม Config Treaty ที่ต้องตรวจสอบ Partner Code (Suthanee.sa 20/11/2024)กรณี Treaty ถูก Config ไว้ให้เลือกเฉพาะรายการตาม Treaty ตาม Partner Code นำ Plan Code ของกรมธรรม์ไปตรวจสอบต่อกรณี Plan Code ถูกกำหนด Package Code ให้เป็น L ให้ประมวลผลเฉพาะ Base LIFEกรณี Plan Code ถูกกำหนด Package Code ให้เป็น Pกรณี RIDER มีประเภทความคุ้มครอง TPD ต้องประมวลผล TPD ไปด้วยเสมอกรณี RIDER มีประเภทความคุ้มครอง ADD ต้องประมวลผล ADD ไปด้วยเสมอกรณี Plan Code ถูกกำหนด Package Code ให้เป็น Nถ้ามี RIDER TPD ตาม Treaty กำหนด ให้ประมวลผล RIDER TPD ด้วยถ้าไม่มี RIDER TPD ตาม Treaty กำหนด ไม่ต้องประมวลผล RIDER ต่อกรณี Treaty ไม่ถูก Config ไว้ ให้ข้ามการตรวจสอบข้อนี้ไป

ตรวจสอบ Effective Date From - To ของ Treaty ที่กำลังประมวลผล
1. กรณีที่ Effective Date To ของ Treaty มากกว่า Period ให้ดำเนินการตรวจสอบ BASE Plan ของแต่ละ Treaty ในขั้นตอนถัดไป (ยังไม่หมดอายุ) (New, Renew, Alter, Claim)
2. กรณีที่ Effective Date To ของ Treaty น้อยกว่า Period (หมดอายุแล้ว) ให้ตรวจสอบ [cf_treaty](http://wiki.thaisamut.co.th/x/-gCaQQ).set_expire กรณีที่ [cf_treaty](http://wiki.thaisamut.co.th/x/-gCaQQ).set_expire = NOTSENTส่งค่า Remark กลับไปอัปเดทว่า "Treaty สิ้นสุดการใช้งาน" (suthanee.sa 24/07/2025) ไม่ต้องดำเนินการประมวลผลใด ๆกรณีที่ cf_treaty.set_expire = SENT ให้ดำเนินการตรวจสอบ BASE Plan ของแต่ละ Treaty ในขั้นตอนถัดไป (Renew, Alter, Claim)กรณีที่ไม่เคยส่ง RE มาก่อน อาจจะเกิดเป็นรายการ New ของปีแรกได้ใน RI Mode รายเดือน หากมีกรมธรรม์ที่ส่ง RE ภายใต้ Treaty นั้นในรูปแบบ New ไปแล้ว ก่อนที่ Treaty นั้นจะหมดอายุ ให้ระบบยังคงคำนวณกรมธรรม์ส่ง New จนครบ 12 เดือน และเข้าสู่เงื่อนไขส่ง Renew ต่อไป ใช้กับทั้ง Base และ Rider***หมายเหตุ : การเทียบ Effective Date To กับ Period ให้เทียบด้วย เดือนกับปี เท่านั้น***
