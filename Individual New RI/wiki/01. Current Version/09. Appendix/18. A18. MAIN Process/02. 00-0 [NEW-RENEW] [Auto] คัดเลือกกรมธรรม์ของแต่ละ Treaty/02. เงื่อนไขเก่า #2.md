การคัดเลือกกรมธรรม์จาก Table Landing ของแต่ละ Treaty (Auto)
หลังจากตรวจสอบ Treaty ที่สามารถนำไปประมวลผลได้แล้ว จะได้ชุดข้อมูล Policy ของแต่ละ Treaty ดังเงื่อนไขต่อไปนี้

Effective From - To ของ Treaty
ตรวจสอบ Effective Date From - To ของ Treaty ที่กำลังประมวลผล
1. กรณีที่ Effective Date To ของ Treaty มากกว่า Period ให้ดำเนินการตรวจสอบ BASE Plan ของแต่ละ Treaty ในขั้นตอนถัดไป (ยังไม่หมดอายุ) (New, Renew, Alter, Claim)
2. กรณีที่ Effective Date To ของ Treaty น้อยกว่า Period (หมดอายุแล้ว) ให้ตรวจสอบ cf_treaty.set_expire กรณีที่ cf_treaty.set_expire = NOTSENTไม่ต้องดำเนินการประมวลผลใด ๆกรณีที่ cf_treaty.set_expire = SENT ให้ดำเนินการตรวจสอบ BASE Plan ของแต่ละ Treaty ในขั้นตอนถัดไป (Renew, Alter, Claim)ใน RI Mode รายเดือน หากมีกรมธรรม์ที่ส่ง RE ภายใต้ Treaty นั้นในรูปแบบ New ไปแล้ว ก่อนที่ Treaty นั้นจะหมดอายุ ให้ระบบยังคงคำนวณกรมธรรม์ส่ง New จนครบ 12 เดือน และเข้าสู่เงื่อนไขส่ง Renew ต่อไป ใช้กับทั้ง Base และ Rider***หมายเหตุ : การเทียบ Effective Date To กับ Period ให้เทียบด้วย เดือนกับปี เท่านั้น***

ตรวจสอบ Rider (ถ้ามี)
เงื่อนไขการคัดเลือกกรมธรรม์ตาม Rider Code ของแต่ละ Treaty

- กรณีที่ Treaty มีการ Config Base Plan Code จะต้องดูเฉพาะกรมธรรม์ที่มี Base Plan Code ตรงกับที่ Config เท่านั้น จึงจะนำ Rider Code ที่กำหนดมากรองกรมธรรม์ที่เข้าเงื่อนไข Treaty นี้ต่อ
- ซึ่งกรณีที่กรมธรรม์ตรงเงื่อนไขเฉพาะ Base Plan Code ก็จะนำไปประมวลผลต่อ แต่ไม่มี Rider Code ส่งไปประมวลผลด้วย
- กรณีที่ไม่มีการ Config Base Plan Code ให้นำ Rider Code ไปหากรมธรรม์ที่มี Rider Code ตรงกับที่ Config มาเข้าเงื่อนไข Treaty นี้ได้เลย

Commencement date
ตรวจสอบ App Date

- ถ้า Commencement date = App Date ให้ใช้ค่า Commencement date
- ถ้า Commencement date <> App Dateกรณี Commencement date < App Date ให้ใช้ค่า App Dateกรณี Commencement date > App Date ให้ใช้ค่า Commencement date (ปกติเงื่อนไขนี้จะไม่มี)

ตรวจสอบ Commencement date/App Date และ Commencement date Rider ทุกครั้ง กรณีที่ Period ที่ประมวลผล กรมธรรม์นั้นยังไม่เกิดขึ้นจริง Commencement date/App Date หรือ Commencement date Rider มากกว่า Period ต้องไม่ประมวลผลรายการ Base หรือ Rider นั้น
