# หลักการคำนวณ Refund (Alter,Claim)

> **Page ID:** 1340768497  
> **Path:** Home / Current Version / 09. Appendix / A18. MAIN Process / 00-1 [ALTER] [Auto] คัดเลือกกรมธรรม์ของแต่ละ Treaty / หลักการคำนวณ Refund (Alter,Claim)

---

**Alter ORD**

1. กรณีที่เป็น Treaty Config ทั้ง Base และ Rider

- ถ้าระบบอ่านเจอรายการที่เป็น LP SD AS ของ Base ก่อน ให้ทำรายการสถานะนี้พร้อมกับ CC Rider ได้เลย (เมื่อทำเสร็จแล้ว ระบบต้อง Mark ว่ารายการ Rider นั้นถูกทำ Refund ไปแล้ว เพื่อไม่ให้ทำรายการซ้ำไม่ว่าจะในกรณีใดก็ตาม)
- ถ้าระบบอ่านเจอรายการที่เป็น CC CD Rider ก่อน ให้ตรวจสอบรายการอื่น ๆ ของกรมธรรมืเดียวกัน ที่มีรอบความคุ้มครองเดียวกัน ว่ามีสถานะ LP SD AS หรือไม่ ถ้ามี ให้ข้ามรายการ CC CD นั้น แล้วไปทำรายการ LP SD AS ที่พบถ้าไม่มี ให้ตรวจสอบรายการ Claim ว่ามีรายการกรมธรรม์เดียวกันที่ Event Date ตรงกับ Alter Eff Date หรือไม่ถ้ามี ให้ข้ามการทำรายการ CC CD นั้น (เป็นการอนุมัติเคลมครบ 100% หรือปฏิเสธเคลมแล้วต้องคืนเงินหรือยกเลิกสัญญา)ถ้าไม่มี ทำรายการ CC CD นั้นได้ตามปกติ (เป็นการ CC CD Rider ปกติ)

2. กรณีที่เป็น Treaty Config เฉพาะ Rider

- ถ้าระบบอ่านเจอรายการที่เป็น LP SD AS ของ Base ก่อน ให้ตรวจสอบรายการอื่น ๆ ของกรมธรรมืเดียวกัน ที่มีรอบความคุ้มครองเดียวกัน ว่ามีสถานะ CC CD หรือไม่ถ้ามี ให้ตรวจสอบรายการ Claim ว่ามีรายการกรมธรรม์เดียวกันที่ Event Date ตรงกับ Alter Eff Date หรือไม่ถ้ามี ให้ข้ามการทำรายการ CC CD นั้น (เป็นการอนุมัติเคลมครบ 100% หรือปฏิเสธเคลมแล้วต้องคืนเงินหรือยกเลิกสัญญา)ถ้าไม่มี ทำรายการ CC CD นั้นได้ตามปกติ (เป็นการ CC CD Rider ปกติ)ถ้าไม่มี ให้ทำรายการ LP SD AS นั้น (เมื่อทำเสร็จแล้ว ระบบต้อง Mark ว่ารายการ Rider นั้นถูกทำ Refund ไปแล้ว เพื่อไม่ให้ทำรายการซ้ำไม่ว่าจะในกรณีใดก็ตาม)
- ถ้าระบบอ่านาเจอรายการที่เป็น CC CD ก่อน ให้ตรวจสอบรายการ Claim ว่ามีรายการกรมธรรม์เดียวกันที่ Event Date ตรงกับ Alter Eff Date หรือไม่ (เมื่อทำเสร็จแล้ว ระบบต้อง Mark ว่ารายการ Rider นั้นถูกทำ Refund ไปแล้ว เพื่อไม่ให้ทำรายการซ้ำไม่ว่าจะในกรณีใดก็ตาม)ถ้ามี ให้ข้ามการทำรายการ CC CD นั้น (เป็นการอนุมัติเคลมครบ 100% หรือปฏิเสธเคลมแล้วต้องคืนเงินหรือยกเลิกสัญญา)ถ้าไม่มี ทำรายการ CC CD นั้นได้ตามปกติ (เป็นการ CC CD Rider ปกติ)

**Alter IND**

1. กรณีที่เป็น Treaty Config เฉพาะ Rider

- ถ้าระบบอ่านาเจอรายการที่เป็น 6 ให้ตรวจสอบรายการ Claim ว่ามีรายการกรมธรรม์เดียวกันที่ Event Date ตรงกับ Alter Eff Date หรือไม่ (เมื่อทำเสร็จแล้ว ระบบต้อง Mark ว่ารายการ Rider นั้นถูกทำ Refund ไปแล้ว เพื่อไม่ให้ทำรายการซ้ำไม่ว่าจะในกรณีใดก็ตาม)ถ้ามี ให้ข้ามการทำรายการ 6 นั้น (เป็นการอนุมัติเคลมครบ 100% หรือปฏิเสธเคลมแล้วต้องคืนเงินหรือยกเลิกสัญญา)ถ้าไม่มี ทำรายการ 6 นั้นได้ตามปกติ (เป็นการ REDUCE PAIDUP (ปิดบัญชี (มีมูลค่า)) ปกติ)

สรุป

- ถ้า Treaty นั้นส่งทั้ง Base และ Rider ระบบจะดูสถานะ LP SD AS เป็นหลักก่อน แล้วค่อยดูสถานะ CC CD
- ถ้า Treaty นั้นส่งเฉพาะ Rider ระบบจะดูสถานะ CC CD เป็นหลักก่อน แล้วจึงดูสถานะ LP SD AS
- ถ้ารายการนั้นเป็น CC CD ต้องเช็กรายการเคลมเสมอเพื่อไม่ให้ทำ Refund ในรอบ Event date นั้น เพราะกรณีที่มีเคลมครบ 100% หรือปฏิเสธเคลมบอกล้าง จะมี Transection CC CD มาเสมอ (ถ้าระบบทำรายการ Alter ก่อน จะได้วันที่ Alter Eff date เท่ากับวันที่เกิดเหตุ ทั้งที่จริง ๆ แล้วในรอบส่ง Re ของวันที่เกิดเหตุ เราจะไม่คืนเงินเลย นอกจากจะเป็นการบอกล้างทั้งสัญญา)

**แต่จากเงื่อนไขทั้งหมด จะต้องไม่เกิดรายการ Refund ซ้ำ  เงื่อนไขข้างต้น เป็นเงื่อนไขในกรณีที่พบรายการ Alteration มาพร้อมกันในรอบเดียวกัน ในกรณีที่มาคนละรอบ ระบบพิจารณาตามเงื่อนไขเดิม และถ้าเคยมีการทำ Refund รายการใดไปแล้วจากเงื่อนไขใหม่ ระบบจะไม่ทำ Refund รายการนั้นซ้ำอีก

**Claim**

1. ในกรณีที่มีการทำรายการเคลมครบ 100% ที่รายการใด ระบบต้อง Mark ว่ารายการ Rider นั้นถูกทำ Refund ไปแล้ว เพื่อไม่ให้ทำรายการซ้ำไม่ว่าจะในกรณีใดก็ตาม แม้จะไม่ได้เป็นการ Refund จริง แต่ต้อง Mark เพื่อไม่ให้รายการอื่นมาทำซ้ำ)
