## SIT Entry Criteria

| No. | Entry Criteria |
| --- | --- |
| 1 | Project plan, and resource allocation must be in placed, and agreed by stakeholders. |
| 2 | Requirement document has been sign-off/acknowledge by users. |
| 3 | In-scope functionality completely delivered based on requirements. |
| 4 | All developed code must be completely unit tested with completed Unit Test Report signed off by development team. |
| 5 | All client/server test hardware, tools, interface systems, test environments must be in placed. |
| 6 | Environment configuration data has been defined and set up. |
| 7 | SIT test preparation complete. |
| 8 | Agreement with stakeholders on the following area:Defect logging workflowPriority definitionSLA to turnaround fixing of reported defects/bugsDeployment schedule |

## Test Deliverable

| No. | Test Deliverable | Description |
| --- | --- | --- |
| 1 | Test plan | Describe test plan, type of testing, test activities, etc. |
| 2 | Test cases and test results | Describe test objective, test condition determine whether system works as expected. |
| 3 | Defect/incident log | Results of unexpected results. |
| 4 | Test report | Summarize results of testing. |

## SIT Exit Criteria

| No. | Exit Criteria |
| --- | --- |
| 1 | All planned test scope have been completed tested. |
| 2 | No outstanding blocker, critical, major, and minor priority/severity defects. |
| 3 | In cases of outstanding defects remain, known issues must be acknowledge and approve by project team. |
