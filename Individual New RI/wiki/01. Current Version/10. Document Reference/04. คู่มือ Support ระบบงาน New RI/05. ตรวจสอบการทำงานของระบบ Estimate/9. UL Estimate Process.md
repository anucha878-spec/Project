4. สามารถตรวจสอบสถานะของรายการประมวลผล เฉพาะ Treaty THREL_UL_RP และ THREL_UL_SP ที่ Column สถานะรายการ ได้ดังนี้

- รอประมวลผล หมายถึงระบบกำลังเข้าคิวรายการเพื่อรอประมวลผล

- รอยืนยันรายการ หมายถึง ระบบประมวลผลรายการนั้นเรียบร้อยแล้ว

- ประมวลผลไม่สำเร็จ หมายถึง การประมวลผลรายการนั้นไม่สำเร็จ (Error) สามารถตรวจสอบ Error ได้ที่หมายเหตุของรายการนั้น

- ไม่สามารถประมวลผลได้ กรุณาติดต่อผู้ดูแลระบบ : ให้กดประมวลผลซ้ำ ครั้งหนึ่ง แต่หากยังประมวลผลไม่ได้ ให้ตรวจสอบ Log การประมวลผลและแจ้งผู้เกี่ยวข้องฝ่าย IT

- มีรายการ Reserve Rate ที่ยังไม่ได้อนุมัติ กรุณาตรวจสอบข้อมูล : ให้ตรวจสอบ Plancode ภายใต้ Treaty นั้นทั้งหมดและตรวจสอบ Reserve Rate ของ Plancode ตามรายการนั้นว่ามีการเพิ่ม Reserve Rate และอนุมัติครบแล้ว

เมื่อเจอ plancode แล้ว ให้แจ้งผู้ใช้งาน (User) ดำเนินการเพิ่มและอนุมัติให้เรียบร้อย จากนั้นจึงกดปุ่มประมวลผลใหม่

5. กรณีที่ถึงเวลาประมวลผลแล้วแต่ระบบยังไม่เริ่มการประมวลผลเลยแม้แต่รายการเดียว ให้ตรวจสอบ Batch ที่เกี่ยวข้อง

- 2. Normal Landing #1 และ 3. Normal Landing #2 ต้อง SUCCESS ทั้งหมด หรือ

- 1. Backup Landing ต้อง SUCCESS ทั้งหมด

กรณี ในกลุ่มข้อใดข้อหนึ่งไม่ SUCCESS ทั้งหมด ให้กด RUN เพื่อให้ทุกรายการ SUCCESS จากนั้นไปที่เมนู ประมวลผลข้อมูล > ดึงข้อมูล as400

![](/download/attachments/1289749801/image2025-10-8%201%3A39%3A4.png?version=1&modificationDate=1759862344176&api=v2)

6. จากนั้นกด RUN RI_AUTO_18 และเลือกประมวลผลรอบ Period ที่ต้องการ จากการยืนยันจาก User

หรือปกติระบบจะต้อง RUN ประมวลผล Period = เดือน -1

- กรณีไม่มีการแจ้งเตือนใด ๆ ให้ไปตรวจสอบการประมวลผลตามข้อที่ 1

- กรณีมีแจ้ง ERROR ให้ตรวจสอบ Log ประมวลผลแล้วแจ้งผู้เกี่ยวข้องฝ่าย IT
