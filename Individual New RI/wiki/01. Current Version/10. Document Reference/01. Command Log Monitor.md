| **ติดตามสถานะของเธรดใน Java process** <![CDATA[sudo watch -n 15 &quot;/usr/lib/jvm/jdk-13.0.1/bin/jstack $(pidof java) \| grep -A 20 &#39;^\&quot;pool-.-thread-1\&quot;.*runnable&#39;&quot;]]> | คำสั่งนี้ใช้สำหรับ **ตรวจสอบสถานะ thread ของ Java application ทุก ๆ 15 วินาที**โดยเฉพาะโฟกัสไปที่ thread ที่ชื่อ `"pool-.-thread-1"` และอยู่ในสถานะ `runnable` พร้อมทั้งแสดง stack trace ของ thread นั้น **เพิ่มเติมอีก 20 บรรทัดถัดไป**`watch -n 15 "..."`: รันคำสั่งซ้ำทุก ๆ 15 วินาที และแสดงผลลัพธ์ออกมา (เหมือน refresh console ทุก ๆ 15 วิ) /usr/lib/jvm/jdk-13.0.1/bin/jstack $(pidof java) ใช้ `jstack` เพื่อดึง stack trace ของ process Java ที่กำลังรันอยู่`$(pidof java)` จะดึง process id ของ Java process (สมมติว่ารันอยู่ตัวเดียว)`grep -A 20`: หาข้อความที่ตรงกับ pattern และแสดง **20 บรรทัดถัดไป**สรุปคำสั่งนี้คือ:**ดู stack trace ของ Java thread ที่ชื่อ `"pool-X-thread-1"` (ไม่ว่าจะเป็น pool ไหน) และอยู่ในสถานะ `runnable` ทุก ๆ 15 วินาที** |
| **ตรวจสอบการใช้ GC (Garbage Collection) ของ Java heap memory** <![CDATA[watch -n 15 &#39;sudo -u msa-reinsurance /usr/lib/jvm/jdk-13.0.1/bin/jstat -gc $(pidof java) \| awk &quot;NR==2 { print \$7, \$8 }&quot;&#39;]]> ![](/download/attachments/1260748873/terminal_OC.png?version=1&modificationDate=1750514397991&api=v2) | คำสั่งนี้มีหน้าที่ **ตรวจสอบการใช้ GC (Garbage Collection) ของ Java heap memory** โดยจะแสดงค่าของ `FBC` (Full GC before compaction) และ `FMC` (Full GC after compaction) ของ JVM process ที่รันโดย user `msa-reinsurance` **ทุก ๆ 15 วินาที**`/usr/lib/jvm/jdk-13.0.1/bin/jstat -gc`: เรียกใช้เครื่องมือ `jstat` เพื่อดู **ข้อมูลของ garbage collector**`$(pidof java)`: หาค่า PID ของโปรเซส `java` (เพื่อส่งให้ `jstat`)`jstat -gc` จะแสดงค่า GC หลายคอลัมน์ เช่น Eden, Survivor, Old Gen, Metaspace, จำนวน GC, etc.สรุป:คำสั่งนี้จะ **แสดงจำนวน Full GC และเวลาที่ใช้ไปกับ Full GC** ของ JVM ที่รันโดย user `msa-reinsurance` ทุก 15 วินาทีเหมาะสำหรับ:ตรวจสอบว่ามีการ Full GC บ่อยหรือไม่เฝ้าระวัง performance ของ JVM |
