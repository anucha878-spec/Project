## Process

1. [เช็ค Effective Date From และ Effective Date To ต้องอยู่ในช่วงของ Treaty](/pages/viewpage.action?pageId=1103266503)
2. เช็คว่า effective date from และ effective date to นั้นทับซ้อนกัน (overlap) กับข้อมูลเดิมที่มีอยู่แล้วในตาราง tx_tmp_plancode_hd อยู่แล้วหรือไม่ หากไม่มีให้ข้ามไปทำข้อ 3. หากมีการทับซ้อนให้ return ดังนี้NameTypeExampleresponseCodeString102responseMessageStringINVALID_EFFECTIVE_DATE
3. บันทึกข้อมูลลงตาราง tx_tmp_plancode_hdDB TableDB FieldRequireDescriptionExampletx_tmp_plancode_hdtmp_treaty_idY 123tx_tmp_plancode_hdproduct_groupY fix 'BASE'tx_tmp_plancode_hdname  09/01/2023 00:00:00tx_tmp_plancode_hdeffective_date_fromY 09/01/2023 00:00:00tx_tmp_plancode_hdeffective_date_toY 09/01/2023 00:00:00tx_tmp_plancode_hdimport_file_name  ไฟล์เลขที่กรมธรรม์ 0001.pdftx_tmp_plancode_hdsum_record จำนวนแถวข้อมูลที่อัปโหลด100tx_tmp_plancode_hdcreated_dateY  tx_tmp_plancode_hdcreated_byY  tx_tmp_plancode_hdupdated_date   tx_tmp_plancode_hdupdated_by
4. นำ contentId ไปดาวน์โหลดไฟล์จากระบบ seaweed และอ่านไฟล์ เพื่อนำข้อมูลที่อยู่ในไฟล์บันทึกลงตาราง tx_tmp_plancode_dt (อ่านข้อมูลลง detail เป็นลักษณะ array [ ] เพราะ 1 ไฟล์มีหลาย record)DB TableDB FieldRequireDescriptionExampletx_tmp_base_plancode_dttmp_plancode_hd_idY id ของตาราง tx_tmp_plancode_hd123tx_tmp_base_plancode_dtpolicy_typeYอ่านข้อมูลจากไฟล์ tx_tmp_base_plancode_dtproduct_groupYอ่านข้อมูลจากไฟล์ tx_tmp_base_plancode_dtplan_codeYอ่านข้อมูลจากไฟล์ tx_tmp_base_plancode_dtcreated_dateY  tx_tmp_base_plancode_dtcreated_byY  tx_tmp_base_plancode_dtupdated_date   tx_tmp_base_plancode_dtupdated_by
