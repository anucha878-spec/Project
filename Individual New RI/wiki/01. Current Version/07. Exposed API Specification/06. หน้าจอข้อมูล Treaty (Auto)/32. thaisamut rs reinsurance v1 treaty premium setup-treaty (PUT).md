## Process

1. นำ premByTreatyHdId ไปค้นหาข้อมูลในตาราง tx_tmp_prem_by_treaty_hd ด้วย query นี้ <![CDATA[select * from tx_tmp_prem_by_treaty_hd where tmp_prem_by_treaty_hd_id = :premByTreatyHdId and effective_date_from = :effectiveDateFrom and effective_date_to = :effectiveDateTo]]>
2. กรณีที่ค้นหาข้อมูลเจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input ไม่มีการปรับแก้ไข effective_date ให้ข้ามไปทำข้อ 4. ได้เลย
3. กรณีที่ค้นหาข้อมูลแล้วไม่เจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input มีการปรับแก้ไข effective_date ให้ทำการเช็คดังนี้ให้ทำการเช็ค effective date ที่ input เข้ามาใหม่นั้น อยู่ในช่วงของ effective date ของ treaty หรือไม่ ([เช็ค Effective Date From และ Effective Date To ต้องอยู่ในช่วงของ Treaty](/pages/viewpage.action?pageId=1103266503))ถ้าได้ค่า return เป็น true หมายความว่า effective date ใหม่ อยู่ในช่วงของ effective date ของ treaty ให้ทำข้อ b. ต่อถ้าได้ค่า return เป็น false หมายความว่า effective date ใหม่ ไม่อยู่ในช่วงของ effective date ของ treaty ให้ return ดังนี้NameTypeDescriptionExampleresponseCodeString100responseMessageStringINVALID_EFFECTIVE_DATE ให้ทำการเช็คว่า effective date ที่ input เข้ามาใหม่นั้น มีข้อมูลช่วงวันที่ตรงกันกับข้อมูลอื่นๆอยู่แล้วหรือไม่ ด้วย query นี้ <![CDATA[select * from tx_tmp_prem_by_treaty_hd where tmp_premium_rate_id = :premRateId and tmp_prem_by_treaty_hd_id &lt;&gt; :premByTreatyHdId and ((effective_date_from between :effectiveDateFrom and :effectiveDateTo) or (effective_date_to between :effectiveDateFrom and :effectiveDateTo))]]> กรณีที่ไม่มีข้อมูล แปลว่า ไม่เคยมีช่วง effective date นี้อยู่ สามารถแก้ไขได้ ให้ข้ามไปทำข้อ 4. ได้เลยกรณีที่มีข้อมูล แปลว่า เคยมีช่วง effective date นี้อยู่แล้ว ไม่สามารถแก้ไขข้อมูลได้ ให้ return ดังนี้NameTypeDescriptionresponseCodeString100responseMessageStringINVALID_EFFECTIVE_DATE
4. ให้ทำการลบข้อมูลที่ตาราง tx_tmp_prem_by_treaty_dt  <![CDATA[delete from tx_tmp_prem_by_treaty_dt where tmp_prem_by_treaty_hd_id = :premByTreatyHdId]]>
5. นำ contentId ไปดาวน์โหลดไฟล์ที่ระบบ seaweed และอ่านไฟล์เพื่อบันทึกข้อมูลลงตาราง tx_tmp_perm_by_traty_dtDB FieldSourcetmp_prem_by_treaty_hd_idpremByTreatyHdIdgenderอ่านข้อมูลจากไฟล์ที่อัพโหลดageอ่านข้อมูลจากไฟล์ที่อัพโหลดprem_rateอ่านข้อมูลจากไฟล์ที่อัพโหลดcreated_datenow()created_byuserLogin
6. อัพเดทข้อมูลตาราง tx_tmp_perm_by_traty_hd ด้วยเงื่อนไข tmp_prem_by_treay_hd_id = :premByTreatyHdIdDB FieldSourcepolicy_typepolicyTypeeffective_date_fromeffectiveDateFromeffective_date_toeffectiveDateToimport_file_namefileNamesum_recordนับจำนวนข้อมูลที่อ่านได้จากไฟล์ที่อัพโหลดupdated_datenow()updated_byuserLogin
