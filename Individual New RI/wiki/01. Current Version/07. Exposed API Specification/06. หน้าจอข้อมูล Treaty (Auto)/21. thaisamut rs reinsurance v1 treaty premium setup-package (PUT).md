## Process

1. นำ premByPackageHdId ไปค้นหาข้อมูลในตาราง tx_tmp_prem_by_package_hd ด้วย query นี้ <![CDATA[select * from tx_tmp_prem_by_package_hd where tmp_prem_by_package_hd_id = :premByPackageHdId and effective_date_from = :effectiveDateFrom and effective_date_to = :effectiveDateTo]]>
2. กรณีที่ค้นหาข้อมูลเจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input ไม่มีการปรับแก้ไข effective_date ให้ข้ามไปทำข้อ 4. ได้เลย
3. กรณีที่ค้นหาข้อมูลแล้วไม่เจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input มีการปรับแก้ไข effective_date ให้ทำการเช็คดังนี้ให้ทำการเช็ค effective date ที่ input เข้ามาใหม่นั้น อยู่ในช่วงของ effective date ของ treaty หรือไม่ ([เช็ค Effective Date From และ Effective Date To ต้องอยู่ในช่วงของ Treaty](/pages/viewpage.action?pageId=1103266503))ถ้าได้ค่า return เป็น true หมายความว่า effective date ใหม่ อยู่ในช่วงของ effective date ของ treaty ให้ทำข้อ b. ต่อถ้าได้ค่า return เป็น false หมายความว่า effective date ใหม่ ไม่อยู่ในช่วงของ effective date ของ treaty ให้ return ดังนี้NameTypeDescriptionExampleresponseCodeString100responseMessageStringINVALID_EFFECTIVE_DATE ให้ทำการเช็คว่า effective date ที่ input เข้ามาใหม่นั้น มีข้อมูลช่วงวันที่ตรงกันกับข้อมูลอื่นๆอยู่แล้วหรือไม่ ด้วย query นี้ <![CDATA[select h.* from tx_tmp_prem_by_package_hd h inner join tx_tmp_premium_rate r on h.tmp_premium_rate_id = r.tmp_premium_rate_id where r.tmp_treaty_id = :tmpTreatyId and h.tmp_premium_rate_id != :tmpPremiumRateId AND ( (effective_date_from BETWEEN :effectiveDateFrom AND :effectiveDateTo) OR (effective_date_to BETWEEN :effectiveDateFrom AND :effectiveDateTo) ) ]]> กรณีที่ไม่มีข้อมูล แปลว่า ไม่เคยมีช่วง effective date นี้อยู่ สามารถแก้ไขได้ ให้ข้ามไปทำข้อ 4. ได้เลยกรณีที่มีข้อมูล แปลว่า เคยมีช่วง effective date นี้อยู่แล้ว ไม่สามารถแก้ไขข้อมูลได้ ให้ return ดังนี้NameTypeDescriptionresponseCodeString100responseMessageStringINVALID_EFFECTIVE_DATE
4. ให้ทำการลบข้อมูลที่ตาราง tx_tmp_prem_by_package_dt  <![CDATA[delete from tx_tmp_prem_by_package_dt where tmp_prem_by_package_hd_id = :premByPackageHdId]]>
5. นำ contentId ไปดาวน์โหลดไฟล์ที่ระบบ seaweed และอ่านไฟล์เพื่อบันทึกข้อมูลลงตาราง tx_tmp_prem_by_package_dtDB FieldSourcetmp_prem_by_package_hd_idpremByTreatyHdIdpolicy_typeอ่านข้อมูลจากไฟล์ที่อัพโหลดproduct_groupอ่านข้อมูลจากไฟล์ที่อัพโหลดplan_codeอ่านข้อมูลจากไฟล์ที่อัพโหลดgenderอ่านข้อมูลจากไฟล์ที่อัพโหลดageอ่านข้อมูลจากไฟล์ที่อัพโหลดlife_prem_rateอ่านข้อมูลจากไฟล์ที่อัพโหลดacc_cls1_prem_rateอ่านข้อมูลจากไฟล์ที่อัพโหลดacc_cls2_prem_rateอ่านข้อมูลจากไฟล์ที่อัพโหลดacc_cls3_prem_rateอ่านข้อมูลจากไฟล์ที่อัพโหลดcreated_datenow()created_byuserLogin
