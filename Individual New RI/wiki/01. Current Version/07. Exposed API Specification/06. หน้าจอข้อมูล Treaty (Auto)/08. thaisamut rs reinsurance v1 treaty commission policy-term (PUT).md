## Process

1. นำ commRateId ไปค้นหาข้อมูลในตาราง tx_tmp_comm_by_term ด้วย query นี้ <![CDATA[select * from tx_tmp_comm_by_term where tmp_comm_rate_id = :commRateId and effective_date_from = :effectiveDateFrom and effective_date_to = :effectiveDateTo]]>
2. กรณีที่ค้นหาข้อมูลเจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input ไม่มีการปรับแก้ไข effective_date ให้ข้ามไปทำข้อ 3. ได้เลยกรณีที่ค้นหาข้อมูลแล้วไม่เจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input มีการปรับแก้ไข effective_date ให้ทำการเช็คดังนี้ให้ทำการเช็ค effective date ที่ input เข้ามาใหม่นั้น อยู่ในช่วงของ effective date ของ treaty หรือไม่ ([เช็ค Effective Date From และ Effective Date To ต้องอยู่ในช่วงของ Treaty](/pages/viewpage.action?pageId=1103266503))ถ้าได้ค่า return เป็น true หมายความว่า effective date ใหม่ อยู่ในช่วงของ effective date ของ treaty ให้ทำข้อ 3. ต่อถ้าได้ค่า return เป็น false หมายความว่า effective date ใหม่ ไม่อยู่ในช่วงของ effective date ของ treaty ให้ return ดังนี้NameTypeDescriptionExampleresponseCodeString 100responseMessageString INVALID_EFFECTIVE_DATE
3. ก่อนทำการ insert ลงตาราง tx_tmp_comm_by_term ให้ทำการลบข้อมูลเดิมออกก่อน โดยใช้เงื่อนไข commRateId จากนั้นค่อยทำการบันทึกข้อมูลใหม่ลงไป <![CDATA[delete from tx_tmp_comm_by_term where tmp_comm_rate_id = :commRateId]]>
4. บันทึกข้อมูลลงตาราง tx_tmp_commission_rate DB TableDB FieldSourceExampletx_tmp_commission_ratetmp_treaty_idtreatyId tx_tmp_commission_ratems_comm_typeLookup ที่ [cf_lookup_catalog](http://wiki.thaisamut.co.th/display/RDSINRI/cf_lookup_catalog)parent_id = 1000500YEARtx_tmp_commission_ratecreated_datenow() tx_tmp_commission_ratecreated_byuserLogin tx_tmp_commission_rateupdated_datenow() tx_tmp_commission_rateupdated_byuserLogin
