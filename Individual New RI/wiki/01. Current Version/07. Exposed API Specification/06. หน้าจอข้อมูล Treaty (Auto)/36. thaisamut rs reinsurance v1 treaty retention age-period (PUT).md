## Process

1. นำ tmpRetentAgeHdId ไปค้นหาข้อมูลในตาราง tx_tmp_retent_age_hd ด้วย query นี้ <![CDATA[select * from tx_tmp_retent_age_hd where tmp_retent_age_hd_id = :tmpRetentAgeHdId and effective_date_from = :effectiveDateFrom and effective_date_to = :effectiveDateTo]]>
2. กรณีที่ค้นหาข้อมูลเจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input ไม่มีการปรับแก้ไข effective_date ให้ข้ามไปทำข้อ 4. และ 5. ได้เลย
3. กรณีที่ค้นหาข้อมูลแล้วไม่เจอ หมายความว่า ข้อมูลเก่าและข้อมูลจาก input มีการปรับแก้ไข effective_date ให้ทำการเช็คดังนี้ให้ทำการเช็ค effective date ที่ input เข้ามาใหม่นั้น อยู่ในช่วงของ effective date ของ treaty หรือไม่ (มี method กลางให้เช็คที่ [UtilComponent.validateTreatyEffectiveDate()](http://wiki.thaisamut.co.th/pages/viewpage.action?pageId=1103266503))ถ้าได้ค่า return เป็น true หมายความว่า effective date ใหม่ อยู่ในช่วงของ effective date ของ treaty ให้ทำข้อ b. ต่อถ้าได้ค่า return เป็น false หมายความว่า effective date ใหม่ ไม่อยู่ในช่วงของ effective date ของ treaty ให้ return ดังนี้NameTypeDescriptionExampleresponseCodeString 100responseMessageString INVALID_EFFECTIVE_DATE ทำให้การเช็คว่า effective date ที่ input เข้ามาใหม่นั้น มีข้อมูลช่วงวันที่ตรงกันกับข้อมูลอื่นๆอยู่แล้วหรือไม่ ด้วย query นี้ <![CDATA[select * from tx_tmp_retent_age_hd where tmp_retention_id = :retentionId and tmp_retent_age_hd_id &lt;&gt; :tmpRetentAgeHdId and ((effective_date_from between :effectiveDateFrom and :effectiveDateTo) or (effective_date_to between :effectiveDateFrom and :effectiveDateTo))]]> กรณีที่ไม่มีข้อมูล แปลว่า ไม่เคยมีช่วง effective date นี้อยู่ สามารถแก้ไขได้ ให้ข้ามไปทำข้อ 4. และ 5. ได้เลยกรณีที่มีข้อมูล แปลว่า เคยมีช่วง effective date นี้อยู่แล้ว ไม่สามารถแก้ไขข้อมูลได้ ให้ return ดังนี้NameTypeDescriptionExampleresponseCodeString100responseMessageStringINVALID_EFFECTIVE_DATE
