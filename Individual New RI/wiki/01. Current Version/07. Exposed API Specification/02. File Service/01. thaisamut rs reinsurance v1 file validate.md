## Overview

## Process

1. ตรวจสอบ request.templateFile ว่าตรงกับที่มีใน cf_template_file หรือไม่ a. กรณีไม่ผ่าน ให้ returnNameTypeExampleresponseCodeString116responseMessageStringInvalid template
2. นำ templateId ไปหาใน table cf_template_file  a. ถ้าไม่เจอ ให้ returnNameTypeExampleresponseCodeString111responseMessageStringNot Exist Template Name
3. ตรวจสอบชื่อไฟล์ ต้องเป็นไปตาม template ที่ได้กำหนดไว้ในตาราง cf_template_file กรณีไม่ผ่านเงื่อนไขจะไม่สามารถอัพโหลดไฟล์ได้กรณีไม่ผ่าน ชื่อไฟล์ไม่ถูกต้อง ให้ return NameTypeExampleresponseCodeString112responseMessageStringINVALID_FILE_NAME
4. ตรวจสอบชื่อไฟล์ว่ามีข้อมูลซ้ำกันหรือไม่ โดยตรวจสอบกับตารางที่ต้องการนำไปบันทึกข้อมูล เช่น ต้องการอัพโหลดไฟล์เกี่ยวกับ Base Plan ให้ทำการเช็คชื่อไฟล์ว่ามีอยู่แล้วในตาราง tx_tmp_plancode_hd แล้วหรือไม่ หากมีแล้วไม่สามารถอัพโหลดไฟล์ได้กรณีไม่ผ่าน ชื่อไฟล์ไม่ถูกต้อง ให้ return NameTypeExampleresponseCodeString113responseMessageStringEXIST_FILE
5. ตรวจสอบ treaty จากชื่อ file และ treaty จาก request ต้องตรงกัน ถ้าไม่ตรงกันให้ returnNameTypeExampleresponseCodeString112responseMessageStringINVALID_FILE_NAME
6. ตรวจสอบ treaty โดยนำ treaty code ไปหาใน table tx_tmp_treaty ถ้าไม่เจอให้ returnNameTypeExampleresponseCodeString114responseMessageStringNot Exist Treaty
